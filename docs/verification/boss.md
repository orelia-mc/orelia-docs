# Boss（ボス）の動作確認

対象: `rpg.boss`（[Boss モジュール仕様](../core/boss.md)）。既存の `monsters.yml` エントリの上に、HP閾値でのフェーズ演出・狂暴化ダメージ倍率・周期的アビリティ発動を重ねます。HP/攻撃力/ドロップのロジックは [Monster の確認手順](monster.md) と共通です。

## 前提条件

`bosses.yml` 標準定義：

| ボスID | 元モンスター | フェーズ | 狂暴化 | アビリティ |
|---|---|---|---|---|
| `goblin_king_boss` | `goblin_king`（HP400） | 70%（激怒開始）, 40%（雄叫び） | HP25%以下でダメージ×1.75 | なし |
| `flame_lord_boss` | `flame_lord`（HP600） | 60%, 25% | HP25%以下でダメージ×2.0 | `meteor_slam`（AOE_SLAM）, `fire_barrage`（FIREBALL_BARRAGE） |

## 1. スポーンコマンドの確認

```
/oladmin spawnboss goblin_king_boss
```

期待結果: `MonsterSpawnService.spawn` に委譲され、`goblin_king`（ZOMBIE, HP400, 攻撃力12, 防御5）と全く同じ見た目・基礎ステータスでスポーンし、加えて `BossAbilityCastService` へ登録される（フェーズ・狂暴化・アビリティの対象になる）。

- `BossModule` を経由しない `/oladmin spawn goblin_king`（通常のMonsterコマンド）でスポーンさせた場合は、ボス演出（フェーズ・狂暴化）が発動**しない**ことを比較確認する（ボス演出は `spawnboss` 経由のみ）。

## 2. フェーズ演出の確認（`BossEncounterListener`）

1. `goblin_king_boss` をスポーンし、HPを70%以下まで削る。
2. 半径48ブロック（`BROADCAST_RADIUS`）以内の全プレイヤーに `&c&lゴブリンキングが激怒し始めた！` がブロードキャストされ、EXPLOSIONパーティクルが発火することを確認する。
3. 40%以下までさらに削り、2つ目のフェーズメッセージ `&4&lゴブリンキングが雄叫びを上げる！` が発火することを確認する。
4. 各フェーズは1度しか発火しない（HPが70%と40%の間で往復してもメッセージが重複しない）ことを確認する。
5. フェーズは読み込み時に閾値の降順でソートされるため、削り方（一気に40%以下まで削る等）に関わらず両方のフェーズメッセージが正しい順序で発火することを確認する。

## 3. 狂暴化の確認（`BossEnrageListener`）

1. `goblin_king_boss` のHPを25%以下まで削る。
2. 狂暴化状態になった以降、ボスからプレイヤーへのダメージが通常時の1.75倍になっていることを実測ダメージで確認する（[Status のダメージ計算式](status.md)を踏まえた上で、通常時攻撃力12ベースの理論値と比較）。
3. `flame_lord_boss` でも同様にHP25%以下で2.0倍になることを確認する。

## 4. 周期アビリティの確認（`flame_lord_boss` のみ）

`BossAbilityCastService.tick()` は20tickごとに、索敵範囲 `AGGRO_RANGE=24.0` 内でクールダウン明けのアビリティを最大1つ発動します。

1. `flame_lord_boss` をスポーンし、24ブロック以内に留まる。
2. `meteor_slam`（AOE_SLAM、ダメージ18、半径6、クールダウン12秒）が発動すると、`&c業火の王が地面を叩きつけた！` のメッセージと共に近くのプレイヤーへ直接ダメージが入ることを確認する。
3. `fire_barrage`（FIREBALL_BARRAGE、ダメージ10、半径3、クールダウン8秒）が発動すると、プレイヤーごとに `SmallFireball` が飛んでくることを確認する。着弾時にブロック破壊が起きない（`EntityExplodeEvent` からブロックダメージが除去される仕様）ことも確認する。
4. 同時に発動するアビリティは最大1つ（同tickで2つ同時発火しない）ことを確認する。
5. クールダウン中は同じアビリティが再発動しないことを、発動直後にすぐ範囲内へ留まって確認する（`meteor_slam` なら12秒、`fire_barrage` なら8秒は再発動しない）。
6. 24ブロックより離れるとアビリティが発動しなくなることを確認する。

## 5. 撃破報酬の確認

ボスも `MonsterDropService` を経由するため、[Monster の確認手順](monster.md) の撃破報酬確認と同じ手順で `goblin_king`（経験値200、お金100〜250、`flame_longsword` 15%）・`flame_lord`（経験値800、お金300〜600）のドロップを確認します。ボス固有の追加ドロップロジックは無いことも合わせて確認してください。

## 異常系・エッジケース

- ボスを倒さずにサーバーを再起動/リロードした場合、`BossRuntimeState`（`phasesTriggered`, `enraged`）は永続化されないため、再起動後は演出状態がリセットされる（同じフェーズメッセージが再度発生し得る）ことを確認する。
- 複数のボスを同時にスポーンさせ、フェーズ・狂暴化・アビリティのクールダウンがボスごとに独立して管理されることを確認する。
