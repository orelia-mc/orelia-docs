# Gathering（採取/農業）の動作確認

対象: `rpg.gathering`（新規モジュール、まだ `docs/core/` にページはありません）。鉱石/丸太採取のブロック再生と、農作物の一括植え付け/収穫、両者に共通するレベル制の一括範囲システムを扱います。モジュール登録順は `Job` の直後・`Item` の手前です（`JobService`/`JobManager` に依存し、レベルアップ通知に現在の職業名を使うため）。

## 前提条件

`gathering.yml` 標準定義（抜粋）：

| 種別 | 対象ブロック | クールダウン | 経験値 | 必要レベル |
|---|---|---|---|---|
| MINING | DIAMOND_ORE / DEEPSLATE_DIAMOND_ORE | 300秒 | 50 | 0 |
| MINING | GOLD_ORE | 180秒 | 25 | 0 |
| MINING | IRON_ORE | 120秒 | 15 | 0 |
| MINING | COAL_ORE | 60秒 | 5 | 0 |
| WOODCUTTING | OAK_LOG / SPRUCE_LOG / BIRCH_LOG | 60秒 | 10 | 0 |
| 農業（FARMING） | WHEAT / CARROTS / POTATOES | - | 8 | - |

レベル帯ごとの一括範囲（半径）: Lv1-9→0（1x1, 実質バニラと同じ）、Lv10-19→1（3x3）、Lv20-29→2（5x5）、Lv30-39→3（7x7）、Lv40-50→4（9x9）。レベル上限は50（`leveling.max-level`）、経験値式は `NextXP = 50 * level^1.5 + 100`。

## 1. コマンドの確認

```
/ol gathering
```

期待結果: 「採取/農業レベル: 1 / 一括範囲(半径): 0」のように、現在のレベルと対応する半径が表示される。

## 2. 通常の採取（単発）の確認

1. 装備状態を問わず（ジョブ不問）、`IRON_ORE` を通常どおり（スニークせず）採掘する。
2. バニラ同様にブロックが壊れドロップが出る。
3. 壊れたブロックの位置には一時的に `replace-block`（`IRON_ORE` の場合 `STONE`）が置かれ、`cooldown-seconds`（120秒）後に元の `IRON_ORE` へ自然に戻ることを確認する（`BlockRegenService`、`GatherChunkLoadListener` によりチャンクがアンロードされても再ロード時に正しく処理される）。
4. `/ol gathering` の経験値/レベルが `xp-gain`（15）分だけ増えていることを確認する。
5. `min-level` が設定されたブロック（標準configでは全て0なので通常は発生しない）を、レベル不足の状態で採掘しようとすると「採取レベルが不足しています。(必要Lv: X)」と表示され、ブロックが壊れないことを確認する（`min-level` をテスト用に一時的に上げて確認すると良い）。

## 3. 一括採取（スニーク）の確認

1. レベル10未満（半径0）の状態でスニークしながら `IRON_ORE` の鉱脈を採掘し、通常時と同じく1ブロックしか処理されないことを確認する。
2. レベルを10以上（半径1=3x3）まで上げ、スニークしながら鉱脈内の1ブロックを採掘すると、周囲3x3x3（中心を除く）の範囲内にある同じ `IRON_ORE` も連動して破壊され、それぞれに経験値・再生スケジュールが個別に設定されることを確認する。
3. 伐採（`WOODCUTTING`）でも同様に、スニーク+高レベルで垂直に伸びた原木がまとめて伐採されることを確認する（縦方向にも半径が適用されるため木の幹に有効）。
4. スニークしていない場合は、レベルに関わらず常に単発破壊のみであることを確認する。

## 4. 農業（植え付け/収穫）の確認

1. 種（`WHEAT_SEEDS`/`CARROT`/`POTATO`）を持ってスニークしながら耕地に右クリックすると、その1マスだけでなく周囲の耕地（レベルに応じた半径、水平面のみ）にもまとめて植えられることを確認する。
2. 植え付け数は手持ちの種の個数で頭打ちになること（種が5個なら最大5マスにしか植わらない）を確認する。
3. スニークしていない通常の右クリックでは、バニラ通りその場の1マスだけに植わることを確認する。
4. 完全に育った作物をホーを持ってスニーク+破壊すると、周囲の完全成長した同じ作物もまとめて収穫されること、収穫数がホーの**残り耐久値**で頭打ちになることを確認する。
5. ホーが耐久0に達した場合、そのホーが壊れる（消滅音が鳴る）ことを確認する。
6. まだ成長しきっていない作物は一括収穫の対象に含まれないことを確認する。
7. 収穫のたびに `xp-gain`（8）分の経験値が加算されることを確認する。

## 5. WorldGuard連携（導入時のみ）の確認

WorldGuardを導入している場合、`RegionProtectionService` がリフレクション経由で `WorldGuardPlugin.canBuild(Player, Block)` を呼びます。

1. WorldGuardで保護されたリージョン内（自分に建築権限が無い場所）で採取/農業を試み、単発・一括のどちらも拒否されることを確認する。
2. 保護外では通常どおり動作することを確認する。
3. WorldGuardを導入していない場合は、コンソールに「WorldGuard not found; gathering/farming will not respect region protection.」という情報ログが起動時に出て、region保護は常に許可扱いになることを確認する（エラーではない）。

## 6. レベルアップ通知の確認

レベルアップ時、`GatheringLevelService` はプレイヤーの現在の職業表示名を使った通知メッセージ（「(プレイヤー名):(職業名)のレベルがXに上がりました」）とレベルアップ音を鳴らします。

1. 未就業の状態でレベルアップし、職業名の部分が「無職」になることを確認する。
2. [Job の確認手順](job.md)で職業に就いた状態でレベルアップし、職業名部分がカタカナ表示名（例: 「フェンサー」）になることを確認する。

## 異常系・エッジケース

- レベル上限（50）到達後は経験値を得てもレベルが上がらず、レベルアップ通知も出ないことを確認する。
- 一括範囲の中心ブロックと種類が異なるブロック（例: `IRON_ORE` の隣に `COAL_ORE` がある）が範囲内にあっても、中心と同じ `Material` のブロックだけが巻き込まれることを確認する。
- サーバー再起動をまたいでも、再生待ち（`replace-block` に置き換わったまま）のブロックが `BlockRegenRepository` から正しく復元され、再起動前のクールダウン残り時間で再生されることを確認する。
